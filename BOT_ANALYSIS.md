## ✅ Что работает отлично

**Онбординг:** `/start` → кнопки цели → кнопки уровня → завершение. Чисто, быстро.

**Диалог:** Бот отвечает в стиле тренера — конкретно, без воды, требует действий ("Встаёшь с дивана СЕЙЧАС"). Тон правильный.

**Режим:** `/mode` работает, MAX/LIGHT логика функционирует.

**Стриминг ответов:** `generate_chat_response_streaming` реализован — бот "печатает" в реальном времени.

**4-слойная память (L0–L4):** Архитектурно готова — классификатор тегов (health/food/training), умная загрузка слоёв. Но слои пустые — нет данных для заполнения.
> ✅ АУДИТ: L0 теперь заполняется через расширенный онбординг (age/height). L2 заполняется авто-парсером питания. L3 — парсером упражнений. L4 — scheduled job воскресенье 21:30. Слои больше не пустые при наличии активности пользователя.

**Авто-суммаризация:** Inactivity reset (180 мин) + token budget compression работают.

**Экспорт CSV:** `/export` — выгрузка тренировок и метрик.

## ⚠️ Проблемы (по приоритету)

### 🔴 КРИТИЧНО — Онбординг не собирает базовые данные
> ✅ РЕАЛИЗОВАНО (Фаза 7) — handlers.py: state machine `awaiting_age` → `awaiting_weight` → `awaiting_height` → `awaiting_workout_time`. Данные сохраняются в `memory_athlete` через `upsert_athlete_card`.

**Что есть:** Онбординг спрашивает только цель + уровень.

**Что нужно:** Возраст, вес, рост — без этого L0-карточка пустая, и бот не может давать персонализированные рекомендации по нагрузке и питанию.

**Где это видно в коде:**
```python
# context_builder.py — _build_l0_card()
if surface.get("age"):
    lines.append(f"Возраст: {surface['age']} лет")
if surface.get("height_cm"):
    lines.append(f"Рост: {surface['height_cm']} см")
```
Поля есть — но `get_l0_surface()` возвращает пустые значения, потому что нет механизма их сбора.

### 🔴 КРИТИЧНО — Нет /profile команды
> ✅ РЕАЛИЗОВАНО (Фаза 7) — `cmd_profile` в commands.py (строка ~245). Показывает: имя, цель, уровень, возраст, рост, вес (из metrics), травмы, режим, стрик, последнюю активность.

Пользователь не может видеть что бот знает о нём. Нет возможности обновить данные. Чёрный ящик.

### 🟡 ВАЖНО — Данные из диалога не парсятся в профиль
> 🟡 ЧАСТИЧНО — Структурированные парсеры работают: `parse_workout_from_message`, `parse_metrics_from_message`, `parse_nutrition_from_message`, `parse_exercises_from_message`. Но свободный текст типа "встаю около 10 по Киеву" по-прежнему не парсится в `memory_training.preferred_time`. Остаётся актуальным как улучшение.

Когда Владислав написал "встаю около 10 по Киеву" — бот ответил, но в БД это никуда не сохранилось. `memory_training.preferred_days`, `memory_athlete.wake_time` пустые.

Парсеры `parse_workout_from_message` и `parse_metrics_from_message` работают только для отчётов о тренировке, не для профильных данных.
